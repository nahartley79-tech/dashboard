S19 rematch — automated was manual
The head-to-head rematch signal no longer needs a hand-run lookup. rematch_auto.py builds one SQL query covering every runner on the card, runs it against the normalised runners/races/race_days schema, and converts the results straight into scoring adjustments. Handles the top-3 recording gap (a recorded win counts as finishing ahead of an unrecorded rival). Points: +1 per rival beaten last meeting (cap +2), +1 turnaround when getting 2kg+ relative relief against a horse that won the last meeting, -1 when giving 1.5kg+ more to a conqueror. Net range -1 to +2.
What else v36 settled
The Jul 4–8 losing run was variance, not breakage. Five straight losing days (11 settled bets, 1 winner, -$96) reproduced identically under v31 and v35 gates in replay — the same horses qualified and lost under both. At the model's 26.5% strike rate, 1-or-fewer winners from 11 is roughly a one-in-six event. No rules were changed in response.

Config drift found and closed. The live Jul 4 run used a stale 150% MR1 overlay cap while the model file said 200% — excluding Tengun Tommy, who won at $4.00.
